How the auth chain works

Caesar's PowerShell session has credential.helper = manager set globally. The first time git needs to authenticate against https://github.com, Windows Credential Manager hands back the fine-grained PAT stored under the target git:https://github.com with user git. Caesar never sees the plaintext value, only the slot's existence via cmdkey /list.

$ cmdkey /list:git:https://github.com
Target: LegacyGeneric:target=git:https://github.com
Type:   Generic
User:   git

How this page was published

GitHub Pages was enabled on this repo via the REST API: POST /repos/matokle-sudo/caesar-smoke/pages with a body of {source: {branch: "main", path: "/"}}. The PAT required the Repository → Pages: Read and write scope to succeed — fine-grained PATs do not grant Pages by default.
